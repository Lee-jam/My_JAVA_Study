# 230227
## 쓰레드
```
개념 이해하기
Thread와 대비되는 두가지 Program, Process
Program은 실행할 수 있는 정적인 상태의 파일
Process는 프로그램의 동작
쓰레드는 프로세스가 할당받은 자원의 단위
```
### Single-thread
- 단일 쓰레드
- 문맥 교환(context switch)작업 불필요

### multi-thread
- 멀티 프로세서 활용의 쓰레드(context switch)

### 쓰레드 생성
- Thread 클래스 상속받는 방법
```
start() : 시작(쓰레드를 사용할 수 있는 환경 구성)
run()   : 구현하고자 하는 부분 작성
native method
```
- Runnable 인터페이스를 구현하는 방법
- 쓰레드를 생성하지 않아도 생성되는 메인 쓰레드가 존재 함.

### 우선순위
- 2개 이상의 스레드가 동작할 때 우선순위를 부여하여 우선순위가 높은 스레드에게 실행의 우선권을 부여
```
누가 먼저 실행되어야 할 지 붙었을 때 누가 우선순위가 높은지에 따라 실행되는 것(대립 구도가 존재할 때)이며
쓰레드는 병렬로 처리되고 있음.
```

### Thread 동기화
- 동기화란 하나의 작업이 완전히 완료된 후 다른 작업을 수행
- 비동기식은 하나의 작업 ㅁ여령 이후 바로 다른 작업 명령을 수행
